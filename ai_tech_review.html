Pre-training

What is pre-training?

Goal: next-token prediction.

  • Most modern LLMs are based on the transformer architecture.
  • Pre-training involves feeding the transformer enormous amounts of raw text, and learning to predict the next token.
  • Self-supervised: the "label"/"target" for each position is just the token that actually comes next in the sequence.
  • The result is a base model that excels at predicting the next token in a sequence by "pattern matching" with its training data.
Everything in pre-training reduces to one question: given the tokens so far, what comes next?
Pre-training

Step 0: Tokenization

Tokens are the atomic units fed to a transformer. Text is split into tokens, and each token maps to an integer ID.

The vocabulary of an LLM is the set of all tokens it can input and that it can predict. Typical vocabulary sizes $d_\text{vocab}$ ≈ 50K (GPT-2) to 100K+ (modern models).

Special tokens are used to represent the start of a sequence, the end of a sequence, and padding.

Tokenized text → Tokenization is just chunking text.

Integer IDs → [ 30642, 1634, 318, 655, 16058, 278, 2420, 13 ]

Common tokenization strategies include:

  • Byte-pair encoding (BPE): break text into subwords, then merge the most frequent pairs to create a vocabulary.
  • Word-level tokenization: split text into whole words.
  • Character-level tokenization: split text into individual characters.
After tokenization, a piece of text becomes a sequence $x_1, x_2, \dots, x_t$, where each $x_i$ is an integer $\in [0, d_\text{vocab} - 1]$.

Given the tokens so far, the model predicts what comes next.

The output of the model is a probability distribution over all possible tokens in its vocabulary.
Formally, the model maps the input $x_1, \dots, x_t$ to a $d_\text{vocab}$-dimensional vector in which coordinate $i$ is the probability that token $i$ comes next given the tokens that came before it — the raw scores that come out of the model's final layer (logits $z \in \mathbb{R}^{d_\text{vocab}}$) passed through a softmax:

$$P(x_{t+1} \mid x_1, \dots, x_t) = \mathrm{softmax}(z)$$

Inference = sample one token from this vector, append it, repeat. That loop is autoregression; temperature / top-p tune how adventurous it is.

At inference we read only the probability distributions at the most-recent input token's position, which are the predictions of what should come immediately after the given sequence.
Self-attention
Each position pulls in information from other positions, via query·key dot products (KQV). This is how context flows between tokens.
MLP
An independent per-position transform — where most factual knowledge is thought to live.
Residuals + layer norm
Neural network architectural tricks that stabilize gradients and help gradients flow through deep layers!
The stack emits a full $\mathbb{R}^{d_\text{vocab}}$ distribution at every position, in a single pass — each predicts the token that follows its own prefix. Training uses all of them (one next-token loss per position); inference reads only the last.
Pre-training

How it learns: cross-entropy loss

We want the model to put high probability on the token that actually came next. So we minimize the negative log-likelihood, averaged over every position:

$$\mathcal{L}(\theta) = -\sum_{t} \log P_\theta\!\left(x_{t+1} \mid x_1, \dots, x_t\right)$$
  • $\theta$ is the model's weights, which are updated during training.
  • $P_\theta$ is the model's output distribution
  • $P_\theta(x_{t+1} \mid x_1, \dots, x_t)$ is the probability the model assigns to the true next token $x_{t+1}$ given the input sequence $x_1, \dots, x_t$

Intuition: each term is -log(probability the model assigned to the correct next-token).

  • If the model assigns a lot of probability to the true next token, $P_\theta(x_{t+1} \mid x_1, \dots, x_t) \approx 1$, and $-\log(1) = 0$: no penalty.
  • If the model assigns a low probability to the true next token, then the loss at timestep $t$ is very high.
  • The goal of training is to drive down the model's "surprise" at the real next token.

We know the whole sequence up front, so the targets are just the input shifted left by one:

targetcatsatonthemat

During training, one LLM forward pass can score all 5 predictions in parallel. Inside the attention layers, a causal mask stops each position from "peeking" ahead to the actual tokens that come after it.

The result: fancy autocomplete

After pre-training, the base model is a brilliant next-token predictor — but it is not an assistant. It just continues text in the most plausible way.

Prompt

What is the capital of France?

A base model might just continue…

What is the capital of Germany?
What is the capital of Italy? …

(it saw lists of quiz questions in training)

What we actually want

The capital of France is Paris.
  • The model absorbed enormous knowledge, grammar, style, even some reasoning — purely as a side effect of learning to represent and predict text effectively.
  • But it has no notion of being asked and answering. Teaching that behavior is post-training → Part 2.
